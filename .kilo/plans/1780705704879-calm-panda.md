# Settings Implementation Plan

## Current State Analysis

### Existing Implementation
- `includes/class-gmcq-settings.php` - Basic settings page with limited fields (quiz_slug, max_questions_per_quiz, max_attempts_per_ip_per_day, uninstall_behavior, cache TTLs, backup settings)
- `includes/class-gmcq-helpers.php` - `gmcq_get_default_settings()` with core settings defined, `gmcq_get_setting()` and `gmcq_reset_settings_cache()` implemented

### Missing Elements (per Settings.md)
1. **Default Quiz Settings section** - Only `max_questions_per_quiz` exists in UI
   - Missing: `default_marks`, `default_negative_marks`, `default_time_limit`, `default_pass_percentage`, `default_questions_per_page`, `default_shuffle_questions`, `default_shuffle_answers`, `default_show_explanations`, `default_show_correct_answers`, `default_require_login`

2. **Frontend Settings section** - Only `quiz_slug` and `max_attempts_per_ip_per_day` exist in UI
   - Missing: `results_display`, `allow_guest_attempts`, `show_timer`, `show_navigation`, `allow_answer_change`

3. **Import Settings section** - Not implemented in UI
   - Missing: `max_csv_size_mb`, `max_import_rows`, `import_batch_size`

4. **Search Settings section** - Partially in defaults but not in UI
   - Missing: `search_min_query_length`, `search_cache_ttl`, `search_max_per_page` (in defaults but not rendered in form)

5. **AJAX handlers missing**:
   - `gmcq_reset_settings` - Reset to defaults
   - `gmcq_export_data` - Export all plugin data

6. **Validation** - No input validation per Settings.md validation rules

7. **Data Management UI** - Reset to Defaults and Export All Data buttons not implemented

## Implementation Steps

### Step 1: Add Missing Default Settings
Update `includes/class-gmcq-helpers.php::gmcq_get_default_settings()` to include all Phase 1 settings:
- Default Quiz Settings: `default_marks`, `default_negative_marks`, `default_time_limit`, `default_pass_percentage`, `default_questions_per_page`, `default_shuffle_questions`, `default_shuffle_answers`, `default_show_explanations`, `default_show_correct_answers`, `default_require_login`
- Frontend Settings: `results_display`, `allow_guest_attempts`, `show_timer`, `show_navigation`, `allow_answer_change`
- Import Settings: `max_csv_size_mb`, `max_import_rows`, `import_batch_size`

### Step 2: Update Settings Save Logic
Update `includes/class-gmcq-settings.php::gmcq_save_settings()` to properly handle:
- Boolean/checkbox fields (convert to 0/1)
- Numeric validation per Settings.md rules
- Decimal field handling for marks

### Step 3: Rebuild Settings Form UI
Rewrite `includes/class-gmcq-settings.php::gmcq_render_settings_page()` to match Settings.md layout:
- Default Quiz Settings section (7 fields)
- Frontend Settings section (6 fields + max_attempts)
- Import Settings section (3 fields)
- Search Settings section (3 fields)
- Cache Settings section (4 fields)
- Data Retention section (3 fields)
- Data Management section (uninstall_behavior + Reset/Export buttons)

### Step 4: Add AJAX Handlers
Add to `includes/class-gmcq-settings.php`:
- `gmcq_ajax_reset_settings()` - Reset settings to defaults with confirmation
- `gmcq_ajax_export_data()` - Export all plugin data as JSON/CSV

### Step 5: Add Validation
Implement validation in save function per Settings.md rules:
- Marks > 0
- Negative marks ≥ 0
- Time limit ≥ 0
- Pass % 1-100
- CSV size ≥ 1 MB
- Max rows ≥ 100
- Batch size 10-500
- Cache TTL ≥ 60
- Max attempts ≥ 0
- Search min length ≥ 1
- Search max per page ≥ 10

### Step 6: Add Export Function
Implement `gmcq_export_all_data()` in `includes/class-gmcq-helpers.php` to export:
- All categories, questions, answers, quizzes_meta, question_map, attempts, attempt_answers, imports, settings

### Step 7: Add Export Function
Implement `gmcq_export_all_data()` in `includes/class-gmcq-settings.php` to export:
- All categories, questions, answers, quizzes_meta, question_map, attempts, attempt_answers, imports, settings
